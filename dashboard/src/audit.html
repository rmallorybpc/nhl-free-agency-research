Site Integrity Audit

This site checks its own work

The audit below scans pages and datasets for common quality issues: broken links, missing first-use definitions, stale files, unrendered loading placeholders, and structural omissions. It runs automatically each week and on every update to this repository. You can also run it live in your browser right now.

What is automated and what still needs human judgment

Automated checks are strong at repeatable validation: links, file freshness, row counts, missing metadata, and known edge-case handling. Automated checks are weaker at readability and clarity for non-technical readers. If language is confusing and this audit does not flag it, that is useful feedback.

Open an issue on GitHub if you find a confusing phrase, weak explanation, or context gap that should be improved.
