Methods Appendix

Regression specifications and reproducibility

This page summarizes the exact model commands used in the analysis pipeline. Source-of-truth documentation remains in the repository methods appendix.

Regression specs

Core analysis script: R/04_analysis/02_regression_models.R. Outcome is points percentage change: points_pct_change = points_percentage - prior_season_points_pct. 

model_a_full <- lm(
  points_pct_change ~ total_aav_spent + prior_season_points_pct + season_type + cap_ceiling_change,
  data = panel_full
)

model_b_full <- lm(
  points_pct_change ~ total_mis + prior_season_points_pct + season_type + cap_ceiling_change,
  data = panel_full
)

model_a_restricted <- lm(
  points_pct_change ~ total_aav_spent + prior_season_points_pct + season_type + cap_ceiling_change,
  data = panel_restricted
)

model_b_restricted <- lm(
  points_pct_change ~ total_mis + prior_season_points_pct + season_type + cap_ceiling_change,
  data = panel_restricted
)

Standard errors and p-values are the default OLS standard errors from R lm() + broom::tidy() in Version 1.

Sensitivity checks

  • Exclude COVID seasons. Restricted models remove rows where season_type is covid_shortened or covid_bubble (2020 and 2021).
  • Alternate MIS weighting. Tier weights can be changed in R/03_feature_engineering/01_build_MIS.R, then all outputs are regenerated.
  • QA checks. Script validates row counts, dependent-variable range, and non-finite values before reporting coefficients.

Reproduce outputs

Rscript R/06_reproducibility/01_reproduce_main_results.R
