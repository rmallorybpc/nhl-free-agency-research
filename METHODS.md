## 2) Scope and exclusion rules

Version 1 scope includes unrestricted free-agent (UFA) signings only.

Explicit exclusions:

- Restricted free agents (RFAs)
- Minor-league contract rows
- Same-team re-signings when constructing movement-event features
- Trade-deadline acquisitions
- AHL call-ups
- Entry-level signings

Operational filters are implemented in:

- `R/02_data_cleaning/01_clean_spotrac.R`
- `R/03_feature_engineering/01_build_MIS.R`

## 3) Variable definitions and formulas

### 3.1 Unit of analysis

- Signing-level table for transaction engineering
- Team-season panel for inferential models

Final model sample:

- 252 team-season rows for seasons 2018 through 2025

### 3.2 Outcome variable

The primary dependent variable is season-over-season team change:

- `points_pct_change = points_percentage - prior_season_points_pct`

Implemented in:

- `R/04_analysis/02_regression_models.R`

### 3.3 Movement Impact Score (MIS)

MIS is a position-weighted aggregate of offseason UFA spending:

- `weighted_aav_i = aav_i * tier_weight_i`
- `total_mis_team_season = sum(weighted_aav_i over signings i)`

Version 1 tier mapping and weights:

- Tier 1 (G): 0.35
- Tier 2 (D): 0.25
- Tier 3 (C): 0.20
- Tier 4 (LW/RW): 0.15
- Tier 5 (other/unmatched): 0.05

Implemented in:

- `R/03_feature_engineering/01_build_MIS.R`

### 3.4 Contract value normalization

Contract fields are normalized as numeric currency values during cleaning.

- `contract_value` and `aav` are parsed from string currency fields
- If `aav` is missing or zero and contract value and years exist:
  - `aav = contract_value / contract_years`
  - row is flagged with `aav_imputed = TRUE`
- Rows with non-usable AAV are retained with `data_quality_note` for transparent filtering in downstream steps

Implemented in:

- `R/02_data_cleaning/01_clean_spotrac.R`

### 3.5 Cap and spending variables

Cap-related variables include:

- `cap_ceiling` from year lookup
- `cap_ceiling_change = cap_ceiling_t - cap_ceiling_(t-1)`
- `cap_spending_pct = total_aav_spent / cap_ceiling`
- `heavy_spender = cap_spending_pct > 0.15`

Implemented in:

- `R/03_feature_engineering/02_build_cap_variables.R`

### 3.6 Prior-finish control

The prior-season team strength control is:

- `prior_season_points_pct`

Construction details:

- Prior-season points percentage is joined by team and prior year
- Utah franchise continuity bridge uses ARI prior-year performance for UTA rows
- Expansion/no-prior rows are filled with prior-year league mean

Implemented in:

- `R/03_feature_engineering/04_build_master_panel.R`

## 4) Model specifications and regression tables

Core model script:

- `R/04_analysis/02_regression_models.R`

### 4.1 Full sample models (2018-2025)

Model A (raw spending):

- `points_pct_change ~ total_aav_spent + prior_season_points_pct + season_type + cap_ceiling_change`

Model B (MIS):

- `points_pct_change ~ total_mis + prior_season_points_pct + season_type + cap_ceiling_change`

### 4.2 Restricted sample robustness models (excluding 2020 and 2021)

Model A restricted:

- same as Model A, estimated on non-COVID rows

Model B restricted:

- same as Model B, estimated on non-COVID rows

### 4.3 Output regression tables

- `output/tables/model_a_full_coefficients.csv`
- `output/tables/model_b_full_coefficients.csv`
- `output/tables/model_a_restricted_coefficients.csv`
- `output/tables/model_b_restricted_coefficients.csv`
- `output/tables/model_comparison_summary.csv`

The reported headline coefficient is the prior-finish slope on `prior_season_points_pct` (approximately -0.42, statistically significant across specifications).

## 5) Sensitivity analyses

### 5.1 Excluding COVID-affected seasons

Sensitivity implemented in production pipeline:

- Excludes `season_type` values `covid_shortened` and `covid_bubble`
- Re-estimates both spending and MIS models on restricted sample

Implemented in:

- `R/04_analysis/02_regression_models.R`

### 5.2 Alternate MIS weighting sensitivity

Version 1 baseline uses fixed tier weights in `R/03_feature_engineering/01_build_MIS.R`.

To run alternate MIS weighting sensitivity:

1. Edit `tier_weights` in `R/03_feature_engineering/01_build_MIS.R`
2. Re-run the reproducibility script in Section 6
3. Compare updated outputs in:
   - `output/tables/model_b_full_coefficients.csv`
   - `output/tables/model_b_restricted_coefficients.csv`
   - `output/tables/model_comparison_summary.csv`

## 8) Limitations

- Correlation does not imply causation.
- UFA spending may be endogenous to front-office expectations, injuries, and latent roster context.
- In-season trades, prospect development, and non-UFA roster channels are not explicitly modeled in Version 1.
- Coefficients are conditional on the current specification and variable construction choices, including MIS weighting assumptions.
